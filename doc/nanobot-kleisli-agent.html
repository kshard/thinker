The Kleisli Category as a Formal Abstraction of AI Agent Behaviour

Dmitry Kolesnikov · github.com/kshard/thinker · April 2026

Abstract

Large-language-model (LLM) agents are typically described informally: a bot receives a prompt, calls tools, and produces a reply. This paper gives a categorical account of that intuition. We show that every agent behaviour — sequential composition, reflective self-correction, and plan-then-execute — can be expressed as a morphism in the Kleisli category of the combined error-and-state monad. The resulting algebra has a single primitive morphism (ReAct\text{ReAct}ReAct), three bridging concepts (Arr\text{Arr}Arr, Arrow\text{Arrow}Arrow, Eff\text{Eff}Eff), and three structural combinators (Seq\text{Seq}Seq, Reflect\text{Reflect}Reflect, ThinkReAct\text{ThinkReAct}ThinkReAct) that correspond directly to well-known patterns in multi-agent system design.

1. Introduction

The practical engineering of LLM-backed systems has outpaced its theoretical foundations. Frameworks accumulate ad-hoc combinators — chains, retries, map-reduce sweeps — without a common algebraic language that would explain why some compositions are valid and others are not, or predict how behaviours compose.

Category theory has long served as the mathematics of composition. The Kleisli construction, in particular, provides a canonical way to reason about effectful programs: it turns a monad into a category whose morphisms are programs with effects, and in which composition is exactly monadic bind (≫ ⁣= ⁣\gg\!=\!≫=). This paper applies that machinery to LLM agents operating over a shared mutable blackboard — a typed state record SSS that accumulates knowledge base.

The central claim is:

An LLM agent step is a Kleisli endomorphism S↠SS \twoheadrightarrow SS↠S on the blackboard. Agent composition is Kleisli composition. All standard multi-agent patterns are instances of this single algebraic structure.

2. Background

2.1 The Error Monad with Context

Let Ctx\text{Ctx}Ctx be the abstract execution context. Define the base monad:

M A  ≜  Ctx→Either(ε, A)M\,A \;\triangleq\; \text{Ctx} \to \text{Either}(\varepsilon,\, A)MA≜Ctx→Either(ε,A)

This is the IO\text{IO}IO-monad restricted to errors, with a Reader\text{Reader}Reader component for Ctx\text{Ctx}Ctx. Monadic return and bind are:

ηA(a)  ≜  λγ.  Right(a)\eta_A(a) \;\triangleq\; \lambda \gamma.\; \text{Right}(a)ηA​(a)≜λγ.Right(a)

(m≫ ⁣=k)  ≜  λγ.  case  m(γ)  of  Left(e)⇒Left(e)∣Right(a)⇒k(a)(γ)(m \mathbin{\gg\!=} k) \;\triangleq\; \lambda \gamma.\; \text{case}\; m(\gamma) \;\text{of}\; \text{Left}(e) \Rightarrow \text{Left}(e) \mid \text{Right}(a) \Rightarrow k(a)(\gamma)(m≫=k)≜λγ.casem(γ)ofLeft(e)⇒Left(e)∣Right(a)⇒k(a)(γ)

Bind sequences two computations, short-circuiting on the first error.

2.2 The State Monad Transformer

Lifting MMM with a state component SSS gives:

StateT S M A  =  S→M (A×S)\text{StateT}\,S\,M\,A \;=\; S \to M\,(A \times S)StateTSMA=S→M(A×S)

A value of type StateT S M A\text{StateT}\,S\,M\,AStateTSMA is a program that reads a blackboard SSS, may fail, and returns both a result of type AAA and an updated blackboard. modify\text{modify}modify, get\text{get}get, and put\text{put}put are the standard primitives.

2.3 The Kleisli Category

Given a monad (M,η,≫ ⁣=)(M, \eta, \gg\!=)(M,η,≫=), the Kleisli category Kl(M)\mathbf{Kl}(M)Kl(M) has:

  • objects: types
  • morphisms A↠BA \twoheadrightarrow BA↠B: functions A→M BA \to M\,BA→MB
  • identity on AAA: ηA:A→M A\eta_A : A \to M\,AηA​:A→MA
  • composition (f:A↠B)> ⁣ ⁣= ⁣ ⁣>(g:B↠C):A↠C(f : A \twoheadrightarrow B) \mathbin{>\!\!=\!\!>} (g : B \twoheadrightarrow C) : A \twoheadrightarrow C(f:A↠B)>=>(g:B↠C):A↠C:

x  ↦  f(x)≫ ⁣=gx \;\mapsto\; f(x) \mathbin{\gg\!=} gx↦f(x)≫=g

Composition is associative and respects the identity laws — this is exactly what the monad laws guarantee.

2.4 Lenses as Bidirectional Accessors

A lens from SSS to AAA is a pair (get,put)(\text{get}, \text{put})(get,put) where get:S→A\text{get} : S \to Aget:S→A extracts a component and put:S×A→S\text{put} : S \times A \to Sput:S×A→S replaces it [4, 5]. The put\text{put}put half is the algebraic structure that this paper requires: it makes SSS a carrier for the action of AAA, folding an agent's output back into the blackboard. Curried, put(a):S→S\text{put}(a) : S \to Sput(a):S→S is the action of command aaa on SSS.

When SSS is a product type (a record), the lens for each field are derived automatically — ensuring that the state monad transformer is consistent with the blackboard's layout.

3. The Agent Algebra

3.1 The Bot Morphism

The primitive abstraction is the Bot — a morphism in Kl(M)\mathbf{Kl}(M)Kl(M):

Bot[S,A]  ≜  S→M A\text{Bot}[S, A] \;\triangleq\; S \to M\,ABot[S,A]≜S→MA

It reads the current blackboard SSS, performs LLM reasoning (including all possible side-effectes), and returns a typed result AAA. Two type parameters are necessary: the input type SSS (the blackboard) and the output type AAA (the result, which is generally different from SSS).

3.2 ReAct: The Primitive Realisation of Bot

The definition of Bot[S,A]\text{Bot}[S, A]Bot[S,A] is abstract — it specifies the type of a Kleisli morphism without prescribing how the LLM interaction is carried out. The ReAct (Reason-and-Act) pattern provides the canonical computational realisation.

A ReAct\text{ReAct}ReAct agent encapsulates a cyclic process of thinking (LLM reasoning), acting (tool invocation — side effect), and observing (feeding tool results back into the context) that repeats until the model produces a final answer. Internally, the agent threads a memory — an ordered sequence of observations — and an encoder/decoder pair that mediates between the typed input AAA / output BBB and the LLM's textual protocol. Externally, the entire cycle is collapsed into a single invocation:

ReAct[A,B]  :  Bot[A,B]\text{ReAct}[A, B] \;:\; \text{Bot}[A, B]ReAct[A,B]:Bot[A,B]

The internal loop is defined as a least fixed point over the LLM reply stage. Let Mem\text{Mem}Mem be the memory state, enc:A→M Prompt\text{enc} : A \to M\,\text{Prompt}enc:A→MPrompt the encoder, dec:Reply→M B\text{dec} : \text{Reply} \to M\,Bdec:Reply→MB the decoder, llm:Prompt×Mem→M (Reply×Mem)\text{llm} : \text{Prompt} \times \text{Mem} \to M\,(\text{Reply} \times \text{Mem})llm:Prompt×Mem→M(Reply×Mem) the model call, and tools:Reply→M Prompt\text{tools} : \text{Reply} \to M\,\text{Prompt}tools:Reply→MPrompt the tool-dispatch oracle. Then:

ReAct(a)  ≜  enc(a)≫ ⁣=loop\text{ReAct}(a) \;\triangleq\; \text{enc}(a) \mathbin{\gg\!=} \text{loop}ReAct(a)≜enc(a)≫=loop

loop(p)  ≜  llm(p)≫ ⁣=λ(r,m′). {dec(r)if r.stage=RETURNtools(r)≫ ⁣=loopif r.stage=INVOKELeft(εaborted)otherwise\text{loop}(p) \;\triangleq\; \text{llm}(p) \mathbin{\gg\!=} \lambda (r, m').\ \begin{cases} \text{dec}(r) & \text{if } r.\text{stage} = \text{RETURN} \\ \text{tools}(r) \mathbin{\gg\!=} \text{loop} & \text{if } r.\text{stage} = \text{INVOKE} \\ \text{Left}(\varepsilon_{\text{aborted}}) & \text{otherwise} \end{cases}loop(p)≜llm(p)≫=λ(r,m′). ⎩⎨⎧​dec(r)tools(r)≫=loopLeft(εaborted​)​if r.stage=RETURNif r.stage=INVOKEotherwise​

Because the loop terminates on RETURN\text{RETURN}RETURN or on error, and external timeout is enforced by Ctx\text{Ctx}Ctx, ReAct\text{ReAct}ReAct is a well-defined Kleisli morphism A↠BA \twoheadrightarrow BA↠B.

Lemma 3.2 (ReAct is a Bot). ReAct[A,B]\text{ReAct}[A, B]ReAct[A,B] satisfies Bot[A,B]\text{Bot}[A, B]Bot[A,B]: it is a function A→M BA \to M\,BA→MB.

Proof. enc:A→M Prompt\text{enc} : A \to M\,\text{Prompt}enc:A→MPrompt, loop:Prompt→M B\text{loop} : \text{Prompt} \to M\,Bloop:Prompt→MB (by case analysis on the reply stage, each branch returns M BM\,BMB). Their Kleisli composition enc> ⁣ ⁣= ⁣ ⁣>loop:A→M B\text{enc} \mathbin{>\!\!=\!\!>} \text{loop} : A \to M\,Benc>=>loop:A→MB. □\square□

The role of ReAct\text{ReAct}ReAct in the algebra is foundational: it is the only combinator that actually invokes an LLM. Every other combinator in the hierarchy (Arrow\text{Arrow}Arrow, Seq\text{Seq}Seq, Reflect\text{Reflect}Reflect, ThinkReAct\text{ThinkReAct}ThinkReAct) is a structural composition that takes one or more Bot\text{Bot}Bot values and produces a new Bot\text{Bot}Bot. The ReAct\text{ReAct}ReAct instances are the leaves of every composition tree.

3.3 From Bot to Arrow: The Bridging Problem

A single ReAct\text{ReAct}ReAct agent produces a result of type AAA, but the blackboard has type SSS. To compose agents into a complex task solver we need endomorphisms S→M SS \to M\,SS→MS, not heterogeneous morphisms S→M AS \to M\,AS→MA. The gap between Bot[S,A]\text{Bot}[S, A]Bot[S,A] and Arr[S]\text{Arr}[S]Arr[S] is bridged by an effect — a description of how to fold a bot's output back into the blackboard and then post-process the result.

Eff[S,A]\text{Eff}[S, A]Eff[S,A] is the composite effect that Arrow\text{Arrow}Arrow applies. It has two components:

Lens[S,A]\text{Lens}[S, A]Lens[S,A] is the put\text{put}put half of a lens (§2.4): it folds the agent's output AAA back into the blackboard. Curried, it is A→(S→S)A \to (S \to S)A→(S→S), a left action of AAA on SSS. When SSS is a product type the lens is derived automatically by structural reflection.

Lens[S,A]  ≜  S×A→S\text{Lens}[S, A] \;\triangleq\; S \times A \to SLens[S,A]≜S×A→S

Eval[S]\text{Eval}[S]Eval[S] is a side-effectful hook that runs after the lens has been applied — persistence, validation, guardrails. Its type coincides with that of Arr[S]\text{Arr}[S]Arr[S] (both are S→M SS \to M\,SS→MS), but its role is different: Eval\text{Eval}Eval is a single post-processing step, whereas Arr\text{Arr}Arr is the composite agent that includes the bot call, the lens, and the eval.

Eval[S]  ≜  S→M S\text{Eval}[S] \;\triangleq\; S \to M\,SEval[S]≜S→MS

The effect bundles both into a single record. When Lens\text{Lens}Lens is absent the lens is derived automatically via structural reflection; when Eval\text{Eval}Eval is absent the identity ηS\eta_SηS​ is used.

Eff[S,A]  ≜  Lens[S,A]×Eval[S]\text{Eff}[S, A] \;\triangleq\; \text{Lens}[S, A] \times \text{Eval}[S]Eff[S,A]≜Lens[S,A]×Eval[S]

3.4 The Kleisli Arrow: Arr\text{Arr}Arr and Arrow\text{Arrow}Arrow

With the bridging types in hand, we define the Kleisli arrow — the endomorphism that every agent step must inhabit — this abstraction enforces determisism for probabilistic Bot\text{Bot}Bot behaviour:

Arr[S]  ≜  S→M S\text{Arr}[S] \;\triangleq\; S \to M\,SArr[S]≜S→MS

Lemma 3.3 (Arr–Bot Isomorphism). Arr[S]≅Bot[S,S]\text{Arr}[S] \cong \text{Bot}[S, S]Arr[S]≅Bot[S,S].

Proof. Both are defined as S→M SS \to M\,SS→MS. The isomorphism is the identity function: given f:Arr[S]f : \text{Arr}[S]f:Arr[S], define Prompt(f)(s)≜f(s)\text{Prompt}(f)(s) \triangleq f(s)Prompt(f)(s)≜f(s); conversely, given b:Bot[S,S]b : \text{Bot}[S, S]b:Bot[S,S], define f(s)≜b(s)f(s) \triangleq b(s)f(s)≜b(s). □\square□

This isomorphism makes every arrow directly usable wherever a bot is expected — the algebra is closed.

Arrow\text{Arrow}Arrow is the canonical lift of Bot[S,A]\text{Bot}[S, A]Bot[S,A] into Arr[S]\text{Arr}[S]Arr[S]. It consumes the bridging Eff\text{Eff}Eff to absorb the output type AAA:

Arrow:Bot[S,A]×Eff[S,A]→Arr[S]\text{Arrow} : \text{Bot}[S, A] \times \text{Eff}[S, A] \to \text{Arr}[S]Arrow:Bot[S,A]×Eff[S,A]→Arr[S]

Arrow(b,φ)  ≜  λs.  b(s)≫ ⁣=λa.  φ.Eval ⁣(φ.Lens(s, a))\text{Arrow}(b, \varphi) \;\triangleq\; \lambda s.\; b(s) \mathbin{\gg\!=} \lambda a.\; \varphi.\text{Eval}\!\bigl(\varphi.\text{Lens}(s,\, a)\bigr)Arrow(b,φ)≜λs.b(s)≫=λa.φ.Eval(φ.Lens(s,a))

The two steps are:

  1. Lens put (pure): φ.Lens(s, a)\varphi.\text{Lens}(s,\, a)φ.Lens(s,a) folds the bot's output into the blackboard — a morphism in the ordinary category of types.
  2. Kleisli composition (effectful): φ.Eval(s′)\varphi.\text{Eval}(s')φ.Eval(s′) applies the side-effect — a morphism in Kl(M)\mathbf{Kl}(M)Kl(M).

Together they yield a single Kleisli endomorphism. Arrow\text{Arrow}Arrow is thus a mapping from the hom-set Bot[S,A]\text{Bot}[S, A]Bot[S,A] into EndKl(M)(S)\text{End}_{\mathbf{Kl}(M)}(S)EndKl(M)​(S), the monoid of endomorphisms on SSS.

Lemma 3.4 (Arrow Well-Definedness). For any b:Bot[S,A]b : \text{Bot}[S, A]b:Bot[S,A] and φ:Eff[S,A]\varphi : \text{Eff}[S, A]φ:Eff[S,A], Arrow(b,φ)\text{Arrow}(b, \varphi)Arrow(b,φ) is a valid Kleisli endomorphism S↠SS \twoheadrightarrow SS↠S.

Proof. By the typing of the constituents. b:S→M Ab : S \to M\,Ab:S→MA. Given s:Ss : Ss:S, b(s)b(s)b(s) yields M AM\,AMA. In the success branch, φ.Lens(s,a):S\varphi.\text{Lens}(s, a) : Sφ.Lens(s,a):S since Lens:S×A→S\text{Lens} : S \times A \to SLens:S×A→S. Then φ.Eval:S→M S\varphi.\text{Eval} : S \to M\,Sφ.Eval:S→MS applied to this result yields M SM\,SMS. Composing via bind:

Arrow(b,φ)(s)  =  b(s)≫ ⁣=λa.  φ.Eval(φ.Lens(s,a))\text{Arrow}(b, \varphi)(s) \;=\; b(s) \mathbin{\gg\!=} \lambda a.\; \varphi.\text{Eval}(\varphi.\text{Lens}(s, a))Arrow(b,φ)(s)=b(s)≫=λa.φ.Eval(φ.Lens(s,a))

which has type M SM\,SMS. Therefore Arrow(b,φ):S→M S=Arr[S]\text{Arrow}(b, \varphi) : S \to M\,S = \text{Arr}[S]Arrow(b,φ):S→MS=Arr[S]. □\square□

3.5 Lifting Computations: Lift\text{Lift}Lift

Not every task solving step involves an LLM call. Pure computations, validation, or transformation steps that operate directly on the blackboard are plain functions S→M SS \to M\,SS→MS. The Lift\text{Lift}Lift combinator injects them into the Kleisli arrow type so they compose uniformly with bot-backed steps:

Lift:Eval[S]→Arr[S]\text{Lift} : \text{Eval}[S] \to \text{Arr}[S]Lift:Eval[S]→Arr[S]

Proof. Lift\text{Lift}Lift is the identity on the underlying function type. Lift\text{Lift}Lift is the formal mechanism by which deterministic code (guardrails, enrichment, persistence) enters the Kleisli algebra on equal footing with LLM-backed agents.

4. Compositional Patterns

4.1 Sequential Composition: Seq\text{Seq}Seq

Seq:Arr[S]∗→Arr[S]\text{Seq} : \text{Arr}[S]^{*} \to \text{Arr}[S]Seq:Arr[S]∗→Arr[S]

Seq(f1,f2,…,fn)  ≜  f1> ⁣ ⁣= ⁣ ⁣>f2> ⁣ ⁣= ⁣ ⁣>⋯> ⁣ ⁣= ⁣ ⁣>fn\text{Seq}(f_1, f_2, \ldots, f_n) \;\triangleq\; f_1 \mathbin{>\!\!=\!\!>} f_2 \mathbin{>\!\!=\!\!>} \cdots \mathbin{>\!\!=\!\!>} f_nSeq(f1​,f2​,…,fn​)≜f1​>=>f2​>=>⋯>=>fn​

6. Algebraic Laws and Their Engineering Implications

We collect the key algebraic properties. All proofs appeal only to the three monad laws of MMM.

6.1 Associativity of Seq\text{Seq}Seq

Theorem 6.1. (Arr[S],Seq,ηS)(\text{Arr}[S], \text{Seq}, \eta_S)(Arr[S],Seq,ηS​) is a monoid.

Proof. Associativity is Lemma 4.1. Left and right identity is Lemma 4.2. □\square□

Engineering implication: agentic behaviour can be grouped and re-grouped arbitrarily. A three-stage agent assembled top-down behaves identically to one assembled bottom-up. Refactoring cannot silently break composition order.

Proof. By Lemma 4.2 (right identity) applied to Seq(f,ηS)=f\text{Seq}(f, \eta_S) = fSeq(f,ηS​)=f, then substitution. □\square□

8. Discussion

8.1 Blackboard as a First-Class Concept

The design choice to make the blackboard type SSS explicit in every bot signature — rather than threading implicit context — has algebraic consequences. It makes Bot[S,A]\text{Bot}[S, A]Bot[S,A] a genuine functor from the category of state types to the category of effectful computations, enabling the Map\text{Map}Map combinator and the automatic lens derivation in Arrow\text{Arrow}Arrow.

8.2 Why Three Patterns, Not One

Seq\text{Seq}Seq, Reflect\text{Reflect}Reflect, and ThinkReAct\text{ThinkReAct}ThinkReAct are all built from the same primitive types, but they are categorically distinct:

  • Seq\text{Seq}Seq is a monoid product in EndKl(M)(S)\text{End}_{\mathbf{Kl}(M)}(S)EndKl(M)​(S).
  • Reflect\text{Reflect}Reflect is a bounded iteration with a decision guard in Kl(M)(S)\mathbf{Kl}(M)(S)Kl(M)(S).
  • ThinkReAct\text{ThinkReAct}ThinkReAct is a cross-category traversal bridging Kl(M)(S)\mathbf{Kl}(M)(S)Kl(M)(S) and Kl(M)(T)\mathbf{Kl}(M)(T)Kl(M)(T) via transform/fold.

Both Reflect\text{Reflect}Reflect and ThinkReAct\text{ThinkReAct}ThinkReAct share a common structure: a specialised bot (Bot[S,Vote[S]]\text{Bot}[S, \text{Vote}[S]]Bot[S,Vote[S]] or Bot[S,[T]]\text{Bot}[S, [T]]Bot[S,[T]]) followed by an arrow tail (Arr[S]\text{Arr}[S]Arr[S] or Arr[T]\text{Arr}[T]Arr[T]). The key difference is that ThinkReAct\text{ThinkReAct}ThinkReAct crosses state spaces — the reactor Arr[T]\text{Arr}[T]Arr[T] operates in the inner per-task category while the overall combinator produces Arr[S]\text{Arr}[S]Arr[S] via the gather fold.

All three produce Arr[S]\text{Arr}[S]Arr[S] (or the isomorphic Bot[S,S]\text{Bot}[S, S]Bot[S,S]), making them freely composable with each other. The Think\text{Think}Think combinator provides the bridge between the planner's output type [A][A][A] and the reactor's state type TTT.

8.3 Reflect\text{Reflect}Reflect as a Bounded Fixed Point

The reflection loop is a bounded iteration of the form:

si+1=c((πState∘j)(si))s_{i+1} = c((\pi_{\text{State}} \circ j)(s_i))si+1​=c((πState​∘j)(si​))

This is not a general fixed point (which would require a complete lattice and a monotone function); it is a bounded unfolding of the recurrence, implemented by Kleisli composition inside a structural recursion on the attempt counter (Lemma 4.6). The bound nnn is an engineering safeguard against non-termination due to a misbehaving judge or corrector.

8.4 Vote\text{Vote}Vote and Judge\text{Judge}Judge: Separation of Evaluation and Decision

The decomposition of the reflection loop's input into two separate concerns — evaluating content (Bot[S,A]\text{Bot}[S, A]Bot[S,A]) and rendering a verdict (Eff[Vote[S],A]\text{Eff}[\text{Vote}[S], A]Eff[Vote[S],A]) — via the Judge\text{Judge}Judge combinator is a consequence of the algebra. The raw judge bot need not know about the Vote\text{Vote}Vote type or the accept/reject protocol. The effect φ:Eff[Vote[S],A]\varphi : \text{Eff}[\text{Vote}[S], A]φ:Eff[Vote[S],A] encodes the decision policy separately, promoting reuse: the same underlying evaluator bot can serve different reflection policies by varying only φ\varphiφ.

8.5 ReAct\text{ReAct}ReAct as the Sole Source of Non-Determinism

Every combinator in the hierarchy except ReAct\text{ReAct}ReAct is a deterministic structural transformation: Arrow\text{Arrow}Arrow applies a pure fold and a side-effect; Seq\text{Seq}Seq threads state; Reflect\text{Reflect}Reflect iterates; Think\text{Think}Think scatters; and ThinkReAct\text{ThinkReAct}ThinkReAct traverses and gathers. The only source of non-determinism (and the only component that contacts an external LLM) is ReAct\text{ReAct}ReAct. This separation has a practical consequence: the structural combinators can be tested and verified independently of any language model, and the composition laws (Theorems 6.1–6.4) hold regardless of which model is wired into the ReAct\text{ReAct}ReAct leaves.

9. Conclusion

We have shown that the behaviour of LLM agents over a shared blackboard is faithfully modelled by the Kleisli category of the error-state monad. The three primitive types — Eval[S]\text{Eval}[S]Eval[S] (effectful state transformer), Lens[S,A]\text{Lens}[S, A]Lens[S,A] (lens put), and Eff[S,A]\text{Eff}[S, A]Eff[S,A] (their product) — together with the Arrow\text{Arrow}Arrow lift and the Arr[S]\text{Arr}[S]Arr[S] type, form a minimal and complete algebra for agent step construction.

The canonical computational realisation of Bot\text{Bot}Bot is the ReAct\text{ReAct}ReAct pattern — a cyclic reason-act-observe loop that is the sole source of non-determinism in the algebra. Every other combinator is a deterministic structural transformation.

The three structural combinators — Seq\text{Seq}Seq (monoid composition), Reflect\text{Reflect}Reflect (bounded iteration with Vote\text{Vote}Vote/Judge\text{Judge}Judge), and ThinkReAct\text{ThinkReAct}ThinkReAct (cross-category transform/unfold traversal) — are derivable instances of standard categorical constructions. Lift\text{Lift}Lift, Judge\text{Judge}Judge, Think\text{Think}Think, and Map\text{Map}Map are derived forms that improve ergonomics without extending the core.

All key properties — associativity of Seq\text{Seq}Seq (Theorem 6.1), identity erasure (Corollary 6.2), functor laws for Map\text{Map}Map (Theorem 6.4), and termination of Reflect\text{Reflect}Reflect (Lemma 4.6) — have been proved from the monad laws alone.
